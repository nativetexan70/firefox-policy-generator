# NIST SP 800-171 Firefox Compliance Profiles

This document describes the NIST SP 800-171 Rev. 3 compliance profiles bundled with firefox-policy-generator. These profiles enforce Firefox security controls aligned with NIST SP 800-171 Revision 3 (the current version, released June 2020) for protecting Controlled Unclassified Information (CUI) in nonfederal systems and organizations.

## Overview

**NIST SP 800-171 Rev. 3** (Special Publication 800-171 Revision 3) is a U.S. federal government standard that specifies security requirements for protecting Controlled Unclassified Information (CUI) in information systems used by nonfederal organizations and federal contractors. It defines 15 security function families with 110 security requirements organized by control family.

### Applicable NIST SP 800-171 Security Families

The NIST SP 800-171 Firefox profiles enforce controls from five security families (shown with their full official titles per NIST SP 800-171 Rev. 3):

| Family | Focus | Firefox Controls |
|--------|-------|------------------|
| **AC** - Access Control | Restrict who can access resources and enforce least privilege | Block add-ons (AC-2), lock about:config (AC-3), disable dev tools (AC-6) |
| **IA** - Identification and Authentication | Verify identity and manage authentication mechanisms | Disable Firefox Accounts (IA-2.1), centralized password mgmt (IA-5.1) |
| **SC** - System and Communications Protection | Protect information in transit using cryptographic controls | TLS 1.3 min (SC-8.1), DoH (SC-8.2), managed proxy (SC-7.5) |
| **SI** - System and Information Integrity | Maintain system integrity, monitor for anomalies, and handle information retention | Auto-updates (SI-2), tracking protection (SI-4.1), data sanitization (SI-12) |

## Profile Levels

The NIST SP 800-171 preset includes two profiles based on the impact level of the CUI being handled:

### CUI-Moderate

**Impact Level**: Moderate confidentiality, integrity, and availability impact if compromised.

**Use Cases**:
- Organizations handling CUI with moderate sensitivity
- Contractors processing government data requiring standard protection
- Systems with moderate impact on business operations if compromised

**Key Controls**:
- Blocks all browser extensions by default
- Enforces TLS 1.3 minimum
- Enables DNS-over-HTTPS
- Disables private browsing (prevents circumvention)
- Blocks risky permissions (geolocation, camera, microphone)
- Clears sensitive data on shutdown
- Disables telemetry and studies
- Prevents unauthorized configuration changes (blocks `about:config`)

**Command Line Usage**:

```bash
python -m ffpolicy generate --preset nist_sp_800_171__cui_moderate -o policies.json
python -m ffpolicy export --preset nist_sp_800_171__cui_moderate --target system_linux --elevate
```

**Locked Settings** (User Cannot Override):
- DNS-over-HTTPS: Enabled
- SSL/TLS Certificate Prompt: Always Ask
- Content Blocking: Strict
- Tracking Protection: Enabled (cryptomining, fingerprinting)
- Data Sanitization: On shutdown
- Add-on Installation: Blocked
- Advanced Configuration: Locked

### CUI-High

**Impact Level**: High confidentiality or high integrity impact if compromised.

**Use Cases**:
- Organizations handling highly sensitive government contracts
- Systems processing classified or near-classified information (CUI with high protections)
- Critical infrastructure contractors
- Organizations with significant regulatory or compliance obligations

**Differences from CUI-Moderate**:
- All CUI-Moderate controls plus
- Stricter permission defaults (all permissions blocked)
- All permission categories locked/enforced
- Enhanced audit trail (updates remain auto-enabled for compliance logging)

**Command Line Usage**:

```bash
python -m ffpolicy generate --preset nist_sp_800_171__cui_high -o policies.json
python -m ffpolicy export --preset nist_sp_800_171__cui_high --target system_linux --elevate
```

## Policy Mapping to NIST SP 800-171 Rev. 3 Controls

The following table maps Firefox policies to NIST SP 800-171 Rev. 3 security requirements (using 03.XX.XX requirement numbering from the authoritative NIST SP 800-171r3 control set):

| NIST Requirement | Title | Firefox Policy | Rationale |
|---|---|---|---|
| 03.01.02 | Access Control - Authorization Enforcement | `InstallAddonsPermission` | Enforce approved authorizations; block malicious add-ons that bypass security controls |
| 03.01.02-alt | Access Control - Authorization Enforcement (Configuration) | `BlockAboutConfig` | Enforce that only approved configurations (via policies.json) are active; prevent unauthorized changes |
| 03.01.05 | Access Control - Least Privilege | `DisableDeveloperTools` | Prevent privileged access to browser internals via developer console; enforce least privilege |
| 03.05.03 | Identification & Authentication - Multi-Factor Authentication | `DisableFirefoxAccounts` | Disable cloud sync without MFA; require organization-managed authentication with MFA |
| 03.05.07 | Identification & Authentication - Password Management | `PasswordManagerEnabled` | Disable local storage; use centralized vault enforcing complexity, rotation, and audit policies |
| 03.13.08 | System & Communications Protection - Transmission and Storage Confidentiality | `SSLVersionMin` (TLS 1.3) | Implement NIST-approved cryptography; enforce TLS 1.3 and disable weak cipher suites |
| 03.13.11 | System & Communications Protection - Cryptographic Protection | `DNSOverHTTPS` | Encrypt all DNS queries to prevent ISP/network operator eavesdropping on CUI browsing |
| 03.14.01 | System & Information Integrity - Flaw Remediation | `DisableAppUpdate` (false) | Enable auto-updates for timely security patch deployment; maintain audit logs of all patches |
| 03.14.06 | System & Information Integrity - System Monitoring | `EnableTrackingProtection` | Detect attacks via tracking scripts; block fingerprinting to prevent profiling and information exfiltration |
| 03.14.08 | System & Information Integrity - Information Management and Retention | `SanitizeOnShutdown` | Manage/retain CUI per policy; clear all sensitive data on shutdown to prevent unauthorized access |

## Deployment

### Single System Export

To apply the NIST SP 800-171 profile to a single system:

```bash
# Export for direct deployment
python -m ffpolicy export --preset nist_sp_800_171__cui_moderate \
  --target system_linux --elevate

# Or export to a specific path
python -m ffpolicy export --preset nist_sp_800_171__cui_moderate \
  --target custom --custom-path /etc/firefox/policies/policies.json --elevate
```

### Multi-System Deployment

Organizations typically deploy via centralized management tools:

```bash
# Generate policies.json for distribution
python -m ffpolicy generate --preset nist_sp_800_171__cui_high -o policies.json

# Distribute to endpoints via:
# - Puppet/Ansible/Chef
# - Windows Group Policy (convert to admx)
# - Mobile Device Management (MDM)
# - macOS system configuration profiles
# - Linux package management (create rpm/deb with policies.json)
```

### Organizational Layering

Organizations often layer NIST SP 800-171 as a baseline, then add organization-specific controls:

```bash
# Start with CUI-Moderate baseline
python -m ffpolicy generate --preset nist_sp_800_171__cui_moderate -o baseline.json

# Edit baseline.json to add org-specific controls:
# - Organization-specific proxy settings
# - Approved extension whitelists
# - Custom homepage/search engine
# - Organization certificate pinning

python -m ffpolicy import baseline.json -o policies.yaml
# ... edit policies.yaml to add org-specific settings ...
python -m ffpolicy generate policies.yaml -o final_policies.json
python -m ffpolicy export final_policies.json --target system_linux --elevate
```

## Browser-Based VPN and Split Tunneling Risks

Browser-based VPN solutions (whether Firefox extensions or built-in VPN features) can introduce **split tunneling** vulnerabilities that undermine NIST SP 800-171 controls, particularly for CUI protection:

### Split Tunneling Risk

Split tunneling allows simultaneous connections to both the VPN and the public internet:
- **Some traffic routes through the VPN** (encrypted, protected by organizational controls)
- **Other traffic routes directly to the internet** (unencrypted, bypassing proxy/monitoring)

This bypasses the following critical NIST SP 800-171 controls:
- **03.13.08** (Transmission Confidentiality): CUI may be transmitted unencrypted
- **03.13.01** (Boundary Protection): Traffic evades organizational firewalls/proxies
- **03.13.11** (Cryptographic Protection): DNS queries may leak outside encrypted tunnel
- **03.14.06** (System Monitoring): Unauthorized traffic evades organizational monitoring

### Mitigations

**For CUI-Moderate and CUI-High Environments:**

1. **Disable browser-based VPN extensions** entirely; use system-level VPN with split tunneling disabled
2. **Use the Proxy policy** to route all traffic through organizational proxy/firewall:
   ```yaml
   Proxy:
     Mode: manual
     HTTPProxy: 'proxy.company.com:8080'
     SSLProxy: 'proxy.company.com:8080'
     NoProxy: 'localhost, 127.0.0.1'  # Minimal no-proxy list
     UseHTTPProxyForAllProtocols: true
     Locked: true
   ```
3. **Lock the Proxy policy** to prevent users from bypassing it
4. **Enable DNS-over-HTTPS** with organizational DoH provider, not public resolvers (prevents DNS leakage)
5. **Educate users** that browser VPN extensions do not provide NIST SP 800-171 compliance

**Why Browser VPN Extensions Are Insufficient:**

| Aspect | System VPN | Browser VPN Extension |
|--------|-----------|----------------------|
| Tunnel coverage | All OS traffic | Browser traffic only |
| Split tunneling prevention | OS-level enforcement | Extension-level only; can be disabled |
| Proxy integration | Full OS support | Firefox-only; may conflict with settings |
| DNS encryption | Managed by OS | Depends on extension configuration |
| Audit trail | OS/network logs | Browser history only; easily cleared |
| Organizational control | Full policy enforcement | Limited to Firefox preferences |

**Compliance Assertion:**
- If your CUI environment uses browser-based VPN extensions, NIST SP 800-171 compliance is **not assured** for data transmitted outside the tunnel
- Organizations must verify with their system administrators that a system-level VPN with split tunneling disabled is in use
- The Proxy policy alone is insufficient without system-level VPN or full firewall enforcement

## Compliance Notes

### Limitations of Policies.json

Some NIST SP 800-171 requirements cannot be expressed in Firefox's `policies.json` alone:

| Requirement | Coverage | Additional Steps |
|-------------|----------|------------------|
| System access logs | Partial | Configure Firefox to use org's authentication system; audit login events |
| Data encryption at rest | Not in policies.json | Configure system-level full-disk encryption (BitLocker, LUKS) |
| Session timeout | Partial | Use OS session management; configure `Preferences` idle timeouts |
| Multi-factor authentication | Not in policies.json | Integrate Firefox with org's SSO/MFA (SAML, OAuth) provider |
| Content filtering | Partial | Deploy proxy/firewall rules; use `Proxy` policy for enterprise proxy |
| Vulnerability scanning | Not in policies.json | Use organization's vulnerability scanning tools |

### Version Compatibility

These profiles are tested against Firefox 148+. Older versions may not support all policies (e.g., `IPProtectionAvailable` in 150+). Verify your Firefox version supports required policies:

```bash
python -m ffpolicy validate --preset nist_sp_800_171__cui_moderate \
  --firefox-version 128
# Reports which policies are unsupported in Firefox 128
```

## Customization

Organizations often need to layer additional controls on top of NIST SP 800-171:

### Approved Extensions Whitelist

For the CUI-Moderate profile, whitelist specific approved extensions:

```yaml
# policies.yaml
ExtensionSettings:
  # Default: block all
  '*':
    installation_mode: blocked
  
  # Approved extensions (organization-specific)
  'ublock0@raymondhill.net':
    installation_mode: force_installed
    install_url: https://addons.mozilla.org/firefox/downloads/file/...
  
  'https-everywhere@eff.org':
    installation_mode: force_installed
    install_url: https://addons.mozilla.org/firefox/downloads/file/...
```

### Enterprise Proxy

Add proxy configuration for enterprise networks:

```yaml
Proxy:
  Mode: manual
  HTTPProxy: 'proxy.company.com:8080'
  SSLProxy: 'proxy.company.com:8080'
  NoProxy: 'localhost, 127.0.0.1, *.company.local'
  UseHTTPProxyForAllProtocols: true
  Locked: true
```

### Custom Homepage/Search

Organizations often set a company intranet as the homepage:

```yaml
Homepage:
  URL: 'https://intranet.company.com'
  Locked: true

Preferences:
  'browser.urlbar.suggest.searches': false
  'browser.search.defaultenginename': 'DuckDuckGo'
```

### Certificate Pinning (via Preferences)

For high-security environments, pin to organization certificates:

```yaml
Preferences:
  'security.cert_pinning.enable': true
  'security.certificate_transparency.mode': 1  # require SCTs
```

## FAQ

**Q: Can I use CUI-Moderate for systems handling highly sensitive data?**
A: CUI-Moderate is designed for moderate impact levels. If your CUI has high confidentiality or integrity impact, use CUI-High. When in doubt, use the stricter profile.

**Q: Why is password manager disabled?**
A: The NIST SP 800-171 profile prioritizes credential management through organizational systems (SSO, MFA, credential vaults) rather than browser storage. Organizations typically require centralized credential management for audit and revocation.

**Q: Can I whitelist specific extensions in CUI-Moderate?**
A: Yes, modify the `ExtensionSettings` policy to allow specific extensions by GUID. However, all extensions should be vetted by your security team before whitelisting.

**Q: How often should I update the policies?**
A: Review and update at least annually, or when Firefox introduces new policies, or when NIST updates the control family. Subscribe to Mozilla security advisories and NIST updates.

**Q: Does this profile provide compliance certification?**
A: This profile implements technical controls aligned with NIST SP 800-171. Compliance is a holistic effort including policies, procedures, training, and auditing. Consult your organization's compliance team and legal advisors.
